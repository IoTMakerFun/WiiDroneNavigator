WiiDroneNavigator by Matt Sellars

Scala application to allow single WiiMote+Nunchuk or dual WiiMote+Nunchuk control of the Parrot AR.Drone quadrocopter.

Download

You can also clone the project with Git by running: 

$ git clone git://github.com/mattsellars/WiiDroneNavigator

To Run

First off please take all precautions when flying the drone. I take no responsibility for your safety when using this software to operate your drone. Use of this software is at your own risk.

Right now it's a rough cut, there's no launcher so you can setup the eclipse project by running as follows: 

 $./gradlew eclipse
Then you will need to create a new scala project making sure you "choose from existing source" in the process. You will need the scala ide plugin with eclipse. Once in eclipse run the application from the main class net.ardrone.DroneFlightApplication

Before you run the application do the following:
  • Connect your computer to your AR.Drone and make sure you get an IP address
  • Run scala application net.ardrone.DroneFlightApplication in eclipse

When the application runs you will be in a terminal. Type "help" for commands.

Connecting a WiiMote + Nunchuck

To connect a WiiMote + Nunchuck do the following:

  • Make sure the nunchuck is plugged into the WiiMote
  • Take the battery cover off the WiiMote to expose the small synch button
  • In the application terminal type 'add wiimote' and hit enter then immediately hit the sync button on the back of the WiiMote
  • If all worked you should see a successful sync message and the lights on the front of the WiiMote should remain solid blue.
  • To add a second remote repeate the above steps

Application Terminal Commands

See help screen for button flight information. To test drone communication you can take off land the drone via the terminal input as follows:

Before you take off position the drone on a flat open surface where it is safe to fly, then to take off enter the terminal command: 

> take off


To land the drone enter the terminal command: 
> land
Exiting the applicaiton will issue a land command to the drone.

If you crash your drone it will enter emergency mode and will not respond. When this happens simply pick the drone up and position it for take off again. Then in the terminal type 'recover'(no quotes) and hit enter. If the drone reset from emergency mode successfully the lights below the propellers will turn green. You can then take off again.

To Turn on the video feed from the cameras use the command: 
> video on


In the future I would like to make the application much easier to use. Until then I hope you have fun. Now you can tell your friends... I don't always fly my AR.Drone but when I do I fly with dual-Nunchuck controls.

Known Issues

On OS X I ran into an issue where the JVM couldn't talk to the native bluetooth drivers. It turned out to be because the drivers were 32bit. If you run into a bluetooth connection problem you can try adding the JVM argument '-d32' to your eclipse run configuration arguments.
